# ブログ記事自動生成 運用ルール

cron 経由で毎日 19:00 に Claude Code を非対話モードで起動し、Notion の「ブログネタ」から
1 件を取り出してエビデンスベースのブログ記事(`draft: true`)を自動生成する運用のルール文書です。

ファイル名・フロントマター・本文の基本フォーマットは [`docs/blog-format.md`](./blog-format.md) に従います。
本文書はそれに加えて、自動生成特有の **ネタソース・エビデンス・引用・記事構成・チェックリスト・Git/Notion 運用** を定めます。

---

## ネタソース

- Notion「ブログネタ」ページから取得する。
- ページは 3 セクション構造:
  - `## 未処理`
  - `## 処理中`
  - `## 完了`
- **「未処理」の最上行(箇条書き 1 件)** を処理対象とする。
- 処理フローに応じて行を移動する:

| タイミング | 操作 |
|------------|------|
| 処理開始時 | 「未処理」→「処理中」に移動 |
| 成功時 | 「処理中」→「完了」に移動、日時と記事 URL を追記 |
| 失敗時 | 「処理中」→「未処理」に戻す + エラーメモを付記 |

- 「未処理」が空の場合は処理対象なし。スキップ扱いとする。

---

## エビデンスルール(テーマ別)

記事のテーマに応じて、求められるエビデンスの種類と優先順位が異なる。

### 医学・健康・栄養系 — 論文必須(査読付きジャーナル)

- 優先順位: **メタアナリシス・システマティックレビュー > RCT > 観察研究**
- 政府機関の資料(厚労省「日本人の食事摂取基準」等)も信頼ソースとして可。
- 主なソース: PubMed、Cochrane Review など。

### 認知科学・脳科学・生産性系 — 論文優先、信頼できる二次情報源可

- 大学・研究機関の発信、研究者本人の発信を優先。
- 二次情報源を使う場合は、必ず**元論文まで辿る**こと。

### 技術・エンジニアリング系 — 公式ドキュメント・一次資料優先

- 公式ドキュメント、原著論文(あれば)、開発者のブログ、GitHub。
- **バージョン・日付の明記**が重要。

### ビジネス・市場系 — 信頼できる調査機関の数字

- 矢野経済研究所、富士経済、Statista、政府統計など。
- 個人ブログや出典不明の数字は使わない。

### 共通ルール

- 「と言われている」「らしい」は禁止。**出典明記が原則**。
- 推測は明示する(「筆者の経験では」「データはないが」など)。
- 5 年以上前のデータは新しいものがないか確認し、無ければ「2020 年時点」などと明記する。
- 海外データを日本に適用する時は注意書きを入れる。

## 引用形式(APA ベース)

### 文中引用

- 基本: `(著者姓, 年)`
- 著者 2 名: `(著者A & 著者B, 年)`
- 著者 3 名以上: `(著者A et al., 年)`
- 日本語著者: 厚生労働省(2020)、山田(2023)など
- 機関: 矢野経済研究所(2024)など
- 直接引用は短いフレーズのみ。引用符 + 出典を付ける。

### 末尾の参考文献

- `## 参考文献` セクションを設ける。
- カテゴリ別に分類する:
  - `### 学術論文`
  - `### 公式ドキュメント`
  - `### 政府・公的資料`
  - `### Web記事`
- 各カテゴリ内は**アルファベット順**。
- **DOI/URL は省略しない**。
- 学術論文形式: `著者(年). タイトル. *ジャーナル名, 巻*(号), ページ. DOI`
- Web サイトは「2025年閲覧」のようにアクセス日を付ける。
- 短縮 URL は使わない。

## 記事構成テンプレート

### セクション構成

1. `## はじめに`(200-400字)— テーマ、関心ポイント、わかることの予告
2. `## 背景・課題`(400-800字)— 背景と問題、論文・統計で裏付け
3. `## 本論`(1000-2000字)— h3 で複数小見出し、エビデンスベースで展開
4. `## 実践への応用 / 考察`(400-800字)— エビデンスから何が言えるか、活用法
5. `## まとめ`(200-400字)— キーポイント箇条書き、次のアクション示唆
6. `## 参考文献`(カテゴリ別)

### 分量・メタ情報

- **全体文字数**: 2,500〜4,500 字
- **タイトル**: 60 字以内、具体的、SEO 意識
- **description**: 100-300 字、結論の一部を含める

### トーン

- です・ます調。
- 既存記事(`src/content/blog/` 内)のトーンに揃える。
- 専門用語は初出時に簡単な説明を添える。
- 過度な感嘆符・絵文字なし。硬めの技術記事。
- 主観と客観を明確に分ける。

## 自己チェックリスト(記事保存前)

- [ ] タイトル 60 字以内
- [ ] description 100-300 字
- [ ] フロントマター項目順が `docs/blog-format.md` に従う
- [ ] `draft: true` になっている
- [ ] 全主張に出典がある
- [ ] 参考文献セクションがある
- [ ] DOI/URL が含まれている
- [ ] 一次資料が少なくとも 1 つ(または「論文なし、二次のみ」と明記)
- [ ] 推測部分が明示されている
- [ ] 全体文字数 2,500〜4,500 字
- [ ] 画像参照があれば `/blog-images/` パス

## Git 運用

- **main ブランチに直接コミット・push** する。
- コミットメッセージ: `chore(blog): auto-draft - {記事タイトル}`

## Notion 更新の詳細

### 成功時 — 「完了」セクション追記フォーマット

```
- YYYY-MM-DD HH:MM | {タイトル} → /blog/{slug}
```

### 失敗時 — 「未処理」復元フォーマット

```
- {元のネタテキスト} [前回失敗: YYYY-MM-DD, {エラー概要}]
```
