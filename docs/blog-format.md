# ブログ記事フォーマット規約

`src/content/blog/` に配置するブログ記事ファイルのファイル名・フロントマターの統一ルールをまとめたドキュメントです。

スキーマ定義: [`src/content/config.ts`](../src/content/config.ts)

---

## 1. ファイル名

### 命名規則

```
YYYY-MM-DD-slug.md
```

| 要素 | 説明 | 例 |
|------|------|-----|
| `YYYY-MM-DD` | 公開日（`pubDate` の日付と揃える） | `2026-02-22` |
| `slug` | 記事タイトルを元にしたスラッグ | `nutrisearch-本番リリースまでの道のり...` |
| 拡張子 | 必ず `.md` | `.md` |

### ルール

- **すべて小文字**（英字は lowercase）
- **スペースはハイフン `-`** に置換する
- 日本語をそのまま含めてよい（Decap CMS が自動生成）
- 記号（`×`, `——`, `、` など）はタイトルから引き継がれる場合がある

### 例（実ファイル）

```
2026-02-08-webサイトのセキュリティヘッダーを強化し、世界上位14-の評価を獲得しました.md
2026-02-13-ai時代のプログラミングスクール——変わるべきビジネスモデルと新しい学びの形.md
2026-02-22-nutrisearch-本番リリースまでの道のり――next-js-16-×-nextauth-js-v5-×-stripe-×-wintercms-で作る栄養検索サービス.md
```

> **注意**: Astro は記事の並び順・表示にファイル名ではなく `pubDate` を使用します。
> ファイル名の日付は管理上の目安なので、`pubDate` と揃えておくと混乱がありません。

## 2. フロントマター

記事の先頭に YAML 形式で記述します。**項目の順序も以下で統一**してください。

```yaml
---
title: 記事タイトル
description: 記事の説明文（SEO・一覧表示で使用。100〜300文字程度）
pubDate: 2026-02-22T08:44:00.000+09:00
author: Yuki Tachi
tags:
  - タグ1
  - タグ2
image: /blog-images/image.png
draft: false
---
```

### 項目一覧

| 項目 | 型 | 必須 | デフォルト | 説明 |
|------|-----|:---:|------|------|
| `title` | string | ✅ | — | 記事タイトル |
| `description` | string | ✅ | — | 記事の説明文。一覧・SEO に使用 |
| `pubDate` | date | ✅ | — | 公開日時。ISO 8601 形式 |
| `author` | string | ー | `Yuki Tachi` | 著者名 |
| `tags` | string[] | ー | — | タグの配列 |
| `image` | string | ー | — | アイキャッチ画像パス（`/blog-images/` 配下） |
| `draft` | boolean | ー | `false` | `true` で下書き（公開されない） |

### 各項目の記述ルール

- **`pubDate`**: JST の場合は `2026-02-22T08:44:00.000+09:00`、UTC の場合は `2026-02-22T00:00:00.000Z`。
  どちらも `z.coerce.date()` で正しくパースされる。
- **`image`**: 画像は `public/blog-images/` に置き、パスは `/blog-images/ファイル名` で参照する。
- **`tags`**: YAML のリスト形式（`- タグ名`）で記述。
- **`draft`**: 執筆中は `true`、公開時に `false`。

## 3. 本文（Markdown）

```markdown
## 見出し（h2 から開始）

本文テキスト。

### 小見出し（h3）

- リスト
- リスト

\`\`\`ts
// コードブロックは言語を指定する
const x = 1;
\`\`\`

![代替テキスト](/blog-images/image.png)
```

### ルール

- 見出しは **`##`（h2）から開始**（`#` はタイトルとして使用済み）
- コードブロックは**言語を明示**する
- 画像は `/blog-images/` に配置してパス参照する
- 画像には `alt` テキストを付ける（アクセシビリティ）

## 4. 新規記事の追加方法

### 方法1: Decap CMS（推奨）

1. https://donut-software.com/admin/ にアクセスしてログイン
2. **New Blog** → 記事作成 → **Publish**
3. ファイル名・フロントマターは自動生成される

### 方法2: 直接ファイル作成

1. `src/content/blog/YYYY-MM-DD-slug.md` を作成
2. 上記フォーマットでフロントマター + 本文を記述
3. `git add` → `git commit` → `git push origin main`（自動デプロイ）
