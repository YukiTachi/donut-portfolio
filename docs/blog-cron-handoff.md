# ブログ記事自動生成システム構築タスク

donut-portfolio に、cron経由で毎日19時に動くブログ記事自動生成システムを構築する。
仕様は事前に決定済みなので、以下に従って段階的に実装してほしい。
各Phaseの終わりに必ず確認を取ること。

## システム概要

スマホ(Notion App)で音声入力 → Notion「ブログネタ」ページ
→ FreeBSD cron(毎日19:00) → Claude Code 起動
→ Notion MCPで未処理ネタを1件取得 → Web検索でエビデンス収集
→ 記事生成(draft: true)→ git push → Notion更新 → メール通知

## 既に完了済みの前提条件

- Notion MCP は user スコープで設定済み、OAuth認証完了
- Notion「ブログネタ」ページ作成済み(3セクション: 未処理/処理中/完了)
- Xserver VPSのメールアカウント cron@donut-service.com 作成済み
- msmtp 設定済み、~/.msmtprc に blog-cron アカウント設定済み
- 送信テスト成功確認済み
  - 送信コマンド例:
    ```sh
    echo "Subject: [blog-cron] テスト送信
    本文" | msmtp yuki.tachi@donut-service.com
    ```

## Phase 1: ドキュメント整備(最初に着手)

### 1-1. docs/blog-format.md を新規作成

内容は別途共有する(添付ファイル「ブログ記事フォーマット.md」の内容そのまま、
ファイル名のみ blog-format.md に変更)。
共有されるまで作業を止めて確認すること。

### 1-2. CLAUDE.md の「ブログ記事のフォーマット」セクションを以下に置き換え

```markdown
## ブログ記事のフォーマット

ブログ記事のファイル名・フロントマター・本文の規約は
**必ず docs/blog-format.md に従う**。要点:

- ファイル名: YYYY-MM-DD-slug.md(小文字、ハイフン区切り、日本語可)
- フロントマター項目順: title → description → pubDate → author → tags → image → draft
- pubDate は ISO 8601(JSTなら +09:00 付き)
- 本文は h2 から開始、コードブロック言語明示、画像は /blog-images/

ブログ自動生成(cron運用)については docs/blog-automation.md を参照。
```

既存の CLAUDE.md の他のセクションは変更しないこと。

### 1-3. docs/blog-automation.md を新規作成

以下のセクションを含む詳細な運用ルール文書:

#### ネタソース

- Notion「ブログネタ」ページから取得
- 3セクション構造: `## 未処理` / `## 処理中` / `## 完了`
- 「未処理」の最上行(箇条書き1件)を処理対象とする
- 処理開始時に「未処理」→「処理中」に移動
- 成功時に「処理中」→「完了」に移動、日時とURL追記
- 失敗時に「処理中」→「未処理」に戻す + エラーメモ

#### エビデンスルール(テーマ別)

**医学・健康・栄養系**: 論文必須(査読付きジャーナル)

- メタアナリシス・システマティックレビュー > RCT > 観察研究の優先順位
- 政府機関(厚労省「日本人の食事摂取基準」等)も信頼ソース可
- ソース: PubMed、Cochrane Review など

**認知科学・脳科学・生産性系**: 論文優先、信頼できる二次情報源可

- 大学・研究機関の発信、研究者発信
- 二次情報源を使う場合は元論文を辿ること

**技術・エンジニアリング系**: 公式ドキュメント・一次資料優先

- 公式ドキュメント、原著論文(あれば)、開発者のブログ、GitHub
- バージョン・日付の明記が重要

**ビジネス・市場系**: 信頼できる調査機関の数字

- 矢野経済研究所、富士経済、Statista、政府統計
- 個人ブログや出典不明の数字は使わない

**共通ルール**:

- 「と言われている」「らしい」禁止、出典明記が原則
- 推測は明示する(「筆者の経験では」「データはないが」など)
- 5年以上前のデータは新しいものがないか確認、無ければ「2020年時点」など明記
- 海外データを日本に適用する時は注意書きを入れる

#### 引用形式(APAベース)

**文中引用**:

- 基本: `(著者姓, 年)`
- 著者2名: `(著者A & 著者B, 年)`
- 著者3名以上: `(著者A et al., 年)`
- 日本語著者: 厚生労働省(2020)、山田(2023)など
- 機関: 矢野経済研究所(2024)など
- 直接引用は短いフレーズのみ、引用符 + 出典

**末尾の参考文献**:

- `## 参考文献` セクション
- カテゴリ別に分類: `### 学術論文` / `### 公式ドキュメント` / `### 政府・公的資料` / `### Web記事`
- 各カテゴリ内アルファベット順
- DOI/URLは省略しない
- 学術論文形式: `著者(年). タイトル. *ジャーナル名, 巻*(号), ページ. DOI`
- Webサイトは「2025年閲覧」のようにアクセス日を付ける
- 短縮URLは使わない

#### 記事構成テンプレート

**セクション構成**:

1. `## はじめに`(200-400字)- テーマ、関心ポイント、わかることの予告
2. `## 背景・課題`(400-800字)- 背景と問題、論文・統計で裏付け
3. `## 本論`(1000-2000字)- h3 で複数小見出し、エビデンスベース展開
4. `## 実践への応用 / 考察`(400-800字)- エビデンスから何が言えるか、活用法
5. `## まとめ`(200-400字)- キーポイント箇条書き、次のアクション示唆
6. `## 参考文献`(カテゴリ別)

**全体文字数**: 2,500〜4,500字
**タイトル**: 60字以内、具体的、SEO意識
**description**: 100-300字、結論の一部を含める

**トーン**:

- です・ます調
- 既存記事(`src/content/blog/` 内)のトーンに揃える
- 専門用語は初出時に簡単な説明
- 過度な感嘆符・絵文字なし、硬めの技術記事
- 主観と客観を明確に分ける

#### 自己チェックリスト(記事保存前)

- [ ] タイトル60字以内
- [ ] description 100-300字
- [ ] フロントマター項目順が `docs/blog-format.md` に従う
- [ ] `draft: true` になっている
- [ ] 全主張に出典がある
- [ ] 参考文献セクションがある
- [ ] DOI/URLが含まれている
- [ ] 一次資料が少なくとも1つ(または「論文なし、二次のみ」と明記)
- [ ] 推測部分が明示されている
- [ ] 全体文字数 2,500〜4,500字
- [ ] 画像参照があれば `/blog-images/` パス

#### Git運用

- main ブランチに直接コミット・push
- コミットメッセージ: `chore(blog): auto-draft - {記事タイトル}`

#### Notion更新の詳細

**成功時の「完了」セクション追記フォーマット**:

```
- YYYY-MM-DD HH:MM | {タイトル} → /blog/{slug}
```

**失敗時の「未処理」復元フォーマット**:

```
- {元のネタテキスト} [前回失敗: YYYY-MM-DD, {エラー概要}]
```

## Phase 2: スクリプト作成

### 2-1. scripts/generate-blog.sh

**機能**:

- シェバン: `#!/usr/local/bin/bash` または `#!/bin/sh`
- 開始時刻記録(変数)
- ログファイル: `~/log/blog-cron-YYYY-MM-DD.log` に追記
- Claude Code を非対話モードで起動、`scripts/cron-blog-prompt.md` を渡す
  - `claude --print "$(cat scripts/cron-blog-prompt.md)"` の形式
  - または `-p` フラグの該当オプション(Claude Codeのバージョンに応じて)
- 終了コードで成功/失敗判定
- ログから生成された記事のタイトル・ファイルパス・slug を抽出
- 結果に応じて成功/失敗/スキップメール送信
- メール本文は事前に決めたテンプレートに従う
- 古いログの削除(30日以上前のログをローテーション)

**エラーハンドリング**:

- Claude Code の実行失敗
- git push の失敗
- Notion 更新の失敗(これは記事生成成功扱いだが警告メール)
- ネタなし(「未処理」が空)の検出 → スキップメール

### 2-2. scripts/cron-blog-prompt.md

Claude Code に渡すプロンプト本文。以下を含む:

- このスクリプトはcronから非対話モードで実行される
- 必ず `docs/blog-automation.md` と `docs/blog-format.md` を最初に読む
- Notion MCPで「ブログネタ」ページを取得
- 「未処理」セクションの最上行を抜き出す
- 「処理中」に移動
- Web検索で論文・エビデンス収集
- 記事生成、自己チェックリストで自己レビュー
- `src/content/blog/YYYY-MM-DD-slug.md` に保存(draft: true)
- git add, commit, push(main)
- Notion を「完了」に更新
- 標準出力に処理結果サマリーを出力(シェルスクリプトがパース可能な形式で)
  - 例: `TITLE=... / FILE=... / SLUG=... / EVIDENCE_COUNT_PAPER=N / ...`
- エラー時は `ERROR=...` を出力して非ゼロ終了

## メールテンプレート詳細

### 成功時

**件名**: `[blog-cron] 記事生成成功: {タイトル}`

**本文**:

- 記事情報(タイトル/ファイル/文字数/タグ/draft状態)
- 元ネタ
- 参考文献概要(カテゴリ別件数)
- レビュー手順
- 処理ログ(開始/終了/所要時間)
- Git情報(ブランチ/コミットハッシュ/メッセージ)

### 失敗時

**件名**: `[blog-cron] 記事生成失敗: {エラー概要}`

**本文**:

- 失敗フェーズ(ネタ取得/論文検索/記事生成/フォーマット検証/Git push/Notion更新のどれか)
- 元ネタ
- エラー概要
- エラー詳細(ログ末尾100行)
- 処理状況(Notion・Gitの状態)
- 次のアクション

### スキップ時(ネタなし)

**件名**: `[blog-cron] スキップ: ブログネタなし`

**本文**: 説明とNotionリンク

### 共通

- From: `cron@donut-service.com`
- To: `yuki.tachi@donut-service.com`
- Content-Type: `text/plain; charset=UTF-8`
- 本文末尾に「このメールは FreeBSD EC2 上の blog-cron から自動送信されています。」
